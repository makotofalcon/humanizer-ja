# Humanizer-JA

AI生成の日本語文章から「AI臭さ」を除去し、人間らしい文章に書き直す Claude Code スキル。

[blader/humanizer](https://github.com/blader/humanizer) の日本語版。英語版の25パターンを日本語に適応しつつ、「これにより」「さらに」の接続詞連打、漢語の過剰連続、翻訳調のカタカナ語など**日本語特有のAI臭さパターン**を追加している。

## インストール

### 推奨（Claude Code のスキルディレクトリに直接クローン）

```bash
mkdir -p ~/.claude/skills
git clone https://github.com/makotofalcon/humanizer-ja.git ~/.claude/skills/humanizer-ja
```

### 手動インストール（SKILL.md のみ）

```bash
mkdir -p ~/.claude/skills/humanizer-ja
cp SKILL.md ~/.claude/skills/humanizer-ja/
```

## 使い方

Claude Code で以下のようにスキルを呼び出す：

```
/humanizer-ja

[ここにテキストを貼り付ける]
```

または直接依頼する：

```
この文章を人間らしくして: [テキスト]
```

## 概要

[Wikipedia: Signs of AI writing](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing) ガイドと、日本語圏のブログ・note・Qiita・Twitter/X での議論を横断的に統合。AI文章の「臭み」を**3層構造**で理解し、25+パターンを検出・修正する。

### 3層構造

| 層 | 内容 | 難易度 |
|---|---|---|
| 第1層 | 記号・書式の残骸（コロン、半角スペース、Markdown） | 即座に除去 |
| 第2層 | 語彙・文体の偏り（「これにより」「さらに」の連発、語尾の単調さ） | 意識的な編集で改善 |
| 第3層 | 思考構造の型（ポジションの不在、体験の欠如、予定調和） | 人間が再構成しない限り解消不可 |

### 最重要機能：アンチAIパス

草稿を書いたあとに**「この文章のどこがAI生成っぽく見えるか？」と自問**し、冷酷な読者の目で残っている違和感を特定・修正する。具体的なチェックリスト付き。

## 25+パターン一覧

### 第1層：記号・書式の残骸

| # | パターン | Before | After |
|---|---------|--------|-------|
| 1 | コロンの不自然な多用 | 「重要な要素は以下の通りです：」 | 「重要なのは〜だ」 |
| 2 | Markdown残骸・半角スペース | 「**非常に重要**な取り組み」 | 「重要な取り組みだ」 |
| 3 | 完璧すぎる箇条書き | 全行同じ文字数・同じ語尾 | 自然な揺らぎのあるリスト |
| 4 | 絵文字の装飾 | 🚀💡✅ がリスト先頭 | 絵文字を削除 |

### 第2層：語彙・文体の偏り

| # | パターン | Before | After |
|---|---------|--------|-------|
| 5 | 「これにより」（最大マーカー） | 「これにより効率が向上しました」 | 「おかげで速くなった」 |
| 6 | 接続詞の連打 | 「さらに」「また」「加えて」が毎段落 | 接続詞なしで繋ぐ |
| 7 | AI頻出語の乱舞 | 「包括的」「網羅的」「多角的」 | 「全体を見て」「いろんな角度から」 |
| 8 | 漢語の過剰連続 | 「促進を図り醸成を推進し構築」 | 「人を育て、空気を良くして、土台をつくる」 |
| 9 | 文末パターンの固定化 | 「〜です」「〜でしょう」のループ | 語尾を多様化 |
| 10 | 冗長な婉曲表現 | 「〜することができます」 | 「〜できる」 |
| 11 | 重要性の過剰強調 | 「画期的な一歩を刻みました」 | 事実を端的に述べる |
| 11b | 意義づけの付け足し | 「浮き彫りにしており」「示唆しており」 | データを提示して読者に委ねる |
| 12 | 「〜だけでなく、〜も」の多用 | "Not only...but also" の直訳 | 直接的に述べる |
| 13 | 3つ組の法則 | 必ず3つセットで並べる | 自然な数で述べる |
| 14 | 同義語の使い回し | 「取り組み」「施策」「イニシアチブ」 | 同じ語を繰り返す |
| 15 | 曖昧な権威づけ | 「専門家によると」 | 具体的な出典を示す |
| 16 | テンプレ的「課題と展望」 | 「課題はあるものの〜期待されます」 | 具体的な事実を述べる |
| 17 | 宣伝調の美辞麗句 | 「息を呑むような」「豊かな」 | 具体的に描写する |
| 18 | 偽の範囲表現 | 「〜から〜に至るまで」 | 直接列挙する |

### 第3層：思考構造の型

| # | パターン | Before | After |
|---|---------|--------|-------|
| 19 | ポジションを取らない中立性 | 「一概には言えません」 | 主観を述べる |
| 20 | 体験談・固有名詞の不在 | 「ある企業では」 | 「freeeが2023年に」 |
| 21 | 金太郎飴の構成 | 導入→本論→まとめのテンプレ | 「転」のある構成 |
| 22 | チャットボットの残骸 | 「ご参考になれば幸いです」 | 削除 |
| 23 | 過剰な共感・お世辞 | 「素晴らしいご質問ですね！」 | 直接回答する |
| 24 | 過剰なヘッジング | 「可能性があるかもしれない」 | 「ありそうだ」 |
| 25 | テンプレ結論 | 「今後の動向に注目です」 | 具体的な計画や事実 |

## 各モデルの癖

| モデル | 特徴 |
|---|---|
| ChatGPT（GPT系） | 「第一に」「第二に」「最後に」と小論文的に整理 |
| Claude | 過剰な共感「そのお気持ち、よくわかります」 |
| Gemini | 見出し・太字・箇条書きをフル活用 |
